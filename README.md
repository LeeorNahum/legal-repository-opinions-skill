# legal-repository-opinions

Opinionated structure and drafting conventions for repositories of operative legal and business documents: agreements, asset assignments, amendments, dissolutions, and the notes around them.

It covers the document lifecycle (draft to executed to superseded), which version controls, ISO-dated file naming with a closed set of status markers, drafting anatomy (articles, recitals, definitions, boilerplate, signature blocks), redline and protection review, dissolution and release drafting, and export-to-signing discipline including e-signature-ready signing.

These are conventions, not legal advice, and not a substitute for a licensed attorney.

The skill is the rules. A repository's own `AGENTS.md` holds that repo's specifics (parties, status, which version controls); the skill neither ships nor assumes one.

## Files

- `SKILL.md` - the conventions, loaded by the agent.
- `AGENTS.md` - maintenance contract for editing this skill.
